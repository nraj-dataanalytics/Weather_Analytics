Weather Analytics & Forecasting Dashboard

1. Project Overview

In this individual project, I designed and developed a comprehensive "Weather Analytics Dashboard" using Power BI. The goal was to create a high-fidelity, interactive visualization tool capable of monitoring real-time weather conditions, tracking atmospheric quality, and forecasting trends for multiple geographic locations including Bangalore, Chennai, and Ajmer.

The dashboard serves as a central hub for meteorological data, presenting complex metrics such as UV index, air quality components (PM10, NO2, SO2), and precipitation probability in a user-friendly, visually engaging interface. It demonstrates the ability to transform raw data into actionable insights through effective visual storytelling.

2. Objectives

The primary objectives of this portfolio project were to:

  • Design an Interactive Interface: Create a dashboard that allows users to seamlessly switch between different city views (Bangalore, Chennai, Ajmer).
  • Visualize Multi-Dimensional Data: Effectively display varying data types—including temperature trends, percentage-based humidity, and air quality indices—on a single screen without clutter.
  • Implement Dynamic Theming: Showcase design versatility by implementing context-aware color themes (e.g., Cool Blue for standard days, Warm Orange for sunny/high-temperature scenarios).
  • Demonstrate Power BI Proficiency: Utilize advanced DAX measures, custom tooltips, and data modeling techniques to ensure accurate reporting.

3. Data Sources & Scope

The dataset utilized for this project simulates a live connection to meteorological services. It encompasses structured data points for multiple Indian cities over a rolling 7-day period.

Data Category Metrics Tracked
Core Weather Temperature (°C), Condition (Partly Cloudy, Sunny), Last Updated Timestamp.
Atmospherics Humidity (%), Wind Speed (Kph), Visibility (KM), Pressure (mb), UV Index.
Air Quality Particulate Matter (PM10, PM2.5), Ozone (O3), Sulfur Dioxide (SO2), Carbon Monoxide (CO), Nitrogen Dioxide (NO2).
Forecast 7-Day Temperature Trends, Rain Probability (%), Sunrise/Sunset times.

4. Dashboard Features

The solution features a modular layout designed for clarity and rapid information consumption. Key components include:

4.1 Location & Status Card

A prominent header card displays the selected city, current date, and a large typographic representation of the current temperature. Visual icons indicate the general weather condition (e.g., "Partly Cloudy").

4.2 Trend Forecasting

A spline area chart visualizes the temperature forecast for the upcoming week. This allows users to instantly identify warming or cooling trends. Data points are labeled clearly (e.g., "23.1°C") to provide precise values at a glance.

4.3 Air Quality Overview

A dedicated section breaks down the Air Quality Index (AQI). It utilizes a radial gauge for the overall score (e.g., "Good 17") and detailed KPI cards for specific pollutants like PM10 and NO2. This provides a granular view of environmental health.

4.4 Environmental Metrics Grid

Secondary metrics such as Humidity, Visibility, and Wind Speed are organized in a uniform grid. Each metric is paired with a relevant icon to improve scanability and user recognition.

4.5 Dynamic Visual Themes

To enhance user experience, two distinct visual themes were developed:

  • Deep Blue Theme: Used for Ajmer and Bangalore views, conveying a clean, modern, and professional aesthetic suitable for standard reporting.
  • Warm Coral/Orange Theme: Applied to the Chennai view, reflecting the warmer climate and providing visual variety.

5. Technical Implementation

This project leveraged several technical features within Power BI:

  • Data Modeling: Created a star schema connecting the Fact_Weather table with Dim_City and Dim_Date tables to facilitate accurate filtering.
  • DAX Measures: Wrote complex DAX formulas to calculate averages, format timestamps dynamically (e.g., "Last Updated, 18 Jun"), and determine the "Chances of Rain" percentages.
  • Custom Backgrounds: Designed custom background layouts using Figma and imported them into Power BI to achieve the "glassmorphism" card effect that is not native to standard visuals.
  • Bookmark Navigation: Implemented bookmarks and selection pane logic to toggle between different city views and manage the visibility of location-specific data.

6. Key Insights Derived

Through the visualization of this data, several insights become immediately apparent:

  1. Temperature Variability: Ajmer shows a significantly higher peak temperature (32.8°C) compared to Bangalore (23.1°C), highlighting regional climatic diversity.
  2. Pollution Correlation: Areas with higher industrial activity (simulated) showed spikes in PM10 and NO2 levels, visually flagged in the Air Quality Overview.
  3. Diurnal Cycles: The "Sunrise and Sunset" module provides critical daylight context, essential for planning outdoor activities or logistics.

7. Challenges and Solutions

Challenge 1: Visual Consistency Across Themes

Issue: Changing color palettes from Blue to Orange often resulted in poor contrast for text elements.
Solution: I established a standardized typography hierarchy using white text for dark backgrounds and adjusted opacity levels to ensure readability regardless of the background gradient.

Challenge 2: displaying Dense Data

Issue: Fitting 15+ metrics (Wind, Rain, AQI, Forecast) onto a single screen without overwhelming the user.
Solution: I adopted a "Card-based" layout. Grouping related metrics (e.g., all Air Quality metrics in one black container) created visual separation and reduced cognitive load.
